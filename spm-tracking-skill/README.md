# SPM埋点自动化设计 Skill

一个用于自动化设计SPM埋点方案的Claude Code Skill。

## 概述

本Skill是资深数据产品专家，精通SPM（Super Position Model）位置模型埋点设计。它的核心能力是：

1. 深度理解需求文档，提取可埋点场景
2. 结构化分析Figma设计稿，识别可交互元素
3. 设计符合规范的SPM编码和参数方案
4. 生成完整的埋点文档
5. 持续沉淀和优化规范文档

## 快速开始

### 安装

```bash
# 方式1: 克隆到本地
git clone <repo-url> ~/.claude/skills/spm-tracking

# 方式2: 使用npx skills
npx skills add <owner>/spm-tracking-skill
```

### 使用

调用Skill时提供以下输入：

1. **需求文档**: PRD或需求说明（.md/.txt）
2. **Figma设计稿截图**: 设计稿截图文件（.png/.jpg）

Skill会自动执行以下流程：
1. 深度理解需求
2. 分析设计稿
3. 确认公共参数
4. 设计SPM方案
5. 设计参数方案
6. 生成埋点文档
7. 询问是否沉淀规范

## 核心概念

### SPM四级结构

```
a级(产品).b级(页面).c级(区块).d级(坑位)

示例: shop.product_detail.banner.buy_btn

- a级: 业务线 (shop/cart/user/pay)
- b级: 页面 (product_detail/checkout)
- c级: 区域 (banner/recommend/feeds)
- d级: 元素 (buy_btn/add_cart/favorite)
```

### 事件类型

| 类型 | 说明 |
|------|------|
| pv | 浏览（页面级） |
| exposure | 曝光（元素级） |
| click | 点击 |
| order | 订单 |

### Human in the Loop

在关键节点会主动询问用户确认：
- 需求理解是否正确
- 设计稿分析是否准确
- SPM设计是否符合规范
- 参数设计是否一致
- 是否需要沉淀规范
