# SPM埋点自动化设计 Skill - 技术规格

## 1. Skill基本信息

| 属性 | 值 |
|------|-----|
| Skill名称 | spm-tracking |
| 版本 | v1.1 |
| 作者 | Claude Code |
| 适用场景 | 电商零售、Robotaxi自动驾驶、网约车出行、智能座舱 |
| 前置要求 | 需求文档（设计稿可选） |

## 2. 技能能力矩阵

| 能力 | 描述 | 优先级 |
|------|------|--------|
| 场景选择 | 确认业务场景，适配对应规范模板 | P0 |
| 需求理解 | 深度理解需求文档，提取埋点场景 | P0 |
| 设计稿分析 | 结构化分析Figma截图，识别可交互元素 | P1 |
| SPM设计 | 设计符合规范的SPM编码 | P0 |
| 参数设计 | 设计符合规范的参数方案 | P0 |
| 文档生成 | 生成完整的埋点方案文档 | P0 |
| 规范沉淀 | 维护和更新全局规范文档 | P1 |

## 4. Human Check节点定义

| 节点 | 触发条件 | 确认内容 | 可跳过 |
|------|---------|----------|--------|
| Phase 1 | 需求理解完成后 | 业务流程、页面、指标 | 是 |
| Phase 2 | 设计稿分析完成后 | 页面结构、元素识别 | 是 |
| Phase 4 | SPM设计完成后 | SPM命名、规范一致性 | 是 |
| Phase 5 | 参数设计完成后 | 参数命名、类型、枚举 | 是 |
| Phase 7 | 文档生成完成后 | 是否需要沉淀规范 | 是 |

## 5. 输入输出定义

### 5.1 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 需求文档 | .md/.txt | 是 | PRD或需求说明 |
| Figma截图 | .png/.jpg | 否 | 设计稿截图（有则分析，无则跳过） |
| 现有SPM清单 | .md | 否 | 已有的SPM枚举值 |
| 公共参数表 | .md | 否 | 已有业务参数 |

### 5.2 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 埋点方案文档 | .md | 完整的SPM埋点方案 |
| 更新规范 | .md | 可选的规范文档更新 |

## 7. 进化机制

### 7.1 版本管理

- 语义化版本: MAJOR.MINOR.PATCH
- MAJOR: 架构重大调整
- MINOR: 新增能力
- PATCH: Bug修复

### 7.2 反馈收集

每个需求完成后收集：
1. 理解偏差: 哪些需求理解有误
2. 设计问题: SPM/参数设计问题
3. 规范缺失: 需要补充的规范

### 7.3 优化方向

- [ ] 增加更多示例
- [ ] 优化确认话术
- [ ] 补充更多场景模板
- [ ] 支持更多平台（iOS/Android/H5）
