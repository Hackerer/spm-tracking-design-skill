# SPM Tracking Skill - 长期记忆（脑干）

> 本文件定义静态规则库，相当于"常识"，是不可撼动的准则。

---

## 1. 数据字典

### 1.1 有效意图判定

| 意图类型 | 判定标准 | 处理方式 |
|---------|---------|---------|
| **有效意图** | 明确包含埋点设计相关需求 | 正常执行 7 Phase |
| **模糊意图** | 提及埋点但信息不足 | 询问补充信息 |
| **拒识意图** | 与埋点无关 | 礼貌拒绝，提示适用范围 |

### 1.2 拒识场景

以下情况应礼貌拒绝：
- 非埋点相关的技术咨询
- 数据分析查询（仅埋点设计）
- 竞品埋点分析（可引导为"设计参考"）
- 纯代码开发需求

### 1.3 系统设置

| 设置项 | 值 | 说明 |
|--------|-----|------|
| skill_name | spm-tracking | 技能名称 |
| version | 1.3 | 当前版本 |
| memory_layers | 3 | 记忆层数 |
| auto_reflect | false | 是否自动反思 |

## 2. 埋点协议

### 2.1 固定 Key 定义

所有埋点输出必须包含以下字段：

| Key | 类型 | 必填 | 说明 |
|-----|------|------|------|
| trace_id | string | 是 | 追踪ID，格式: `{日期}_{场景}_{序号}` |
| event_type | string | 是 | 事件类型: pv/exposure/click/order/tech |
| spm | string | 是 | SPM编码，格式: a.b.c.d |
| biz_type | string | 否 | 业务类型 |
| timestamp | number | 是 | 时间戳（毫秒） |
| params | object | 否 | 业务参数 |

### 2.2 输出格式规范（Markdown表格）

#### 2.2.1 埋点清单表格格式

```
浏览事件：

| a | b | b浏览 | 触发时机 | 参数 |
|---|---|------|----------|------|
| 中文 | 中文 | 是/否 | 触发时机描述 | [公共参数] |

曝光点击事件：

| c | c曝光 | d | d截图 | d坑位类型 | d事件类型 | 触发时机 | 参数 |
|---|-------|---|-----|----------|----------|----------|------|
| 中文 | 否 | 中文+英文(英文) | - | 固定坑位/顺序坑位 | 点击/曝光/技术 | 描述 | [公共参数] + 参数 |
```

#### 2.2.2 命名规则

| 列 | 规则 | 示例 |
|----|------|------|
| a | 中文 | 网易云音乐 / 无麦K歌 / HMI基础 |
| b | 中文 | 首页 / 播放器 / 桌面 |
| c | 中文 | 播放器 / 排行榜 / 语音推荐区 |
| d | 中文+英文 | 播放(play_btn) / 排行榜卡片(rank_card) |

#### 2.2.3 重要规则

1. **c 曝光列**: 大部分情况为"否"，因为 d 曝光可以覆盖 c；特殊场景下可设为"是"（如整体区块曝光有独立统计意义时）
2. **d 事件类型**: 根据实际需要标记为"点击"、"曝光"、"技术"等
3. **参数分隔**: 参数之间用空格分隔，**禁止使用逗号**，避免 Markdown 表格解析错误
4. **表格空行**: 标题和表格之间必须有空行分隔

#### 2.2.4 参数格式示例

| 错误格式 | 正确格式 |
|---------|---------|
| `[公共参数] + song_id, action` | `[公共参数] + song_id action` |
| `[公共参数] + rank_id, song_id, index` | `[公共参数] + rank_id song_id index` |

#### 2.2.5 b 类型标注

每个 b（页面/组件）需要标注类型：

| 类型 | 说明 | 浏览事件 |
|------|------|----------|
| 页面 | 有独立浏览事件 | 有浏览事件表格 |
| 组件 | 全局组件，无独立浏览 | 写"浏览事件：无" |

示例：
```
### 4.1 桌面 (b: desktop) - 页面

浏览事件：

| a | b | b浏览 | 触发时机 | 参数 |
|---|---|------|----------|------|
| HMI基础 | 桌面 | 是 | 每次打开桌面时 | [公共参数] |

---

### 4.2 调音弹窗 (b: karaoke_tuning) - 组件

浏览事件：无
```

## 3. 业务边界

### 3.1 场景边界

| 场景类型 | 支持 | 说明 |
|---------|------|------|
| 电商零售 | ✅ | 全流程支持 |
| Robotaxi | ✅ | 自动驾驶出租车 |
| 网约车出行 | ✅ | 快车/专车/出租车 |
| 智能座舱 | ✅ | IVI/ADAS/语音 |
| 金融理财 | ✅ | 账户/交易/理财 |
| 在线教育 | ✅ | 课程/直播/学习 |
| 工具类App | ✅ | 功能性工具 |
| 其他 | ⚠️ | 需用户确认 |

### 3.2 特殊规则

| 业务类型 | 规则 | 说明 |
|---------|------|------|
| **金融类埋点** | 必须标注"敏感" | 参数中包含金额/账户需标注 |
| **车控类埋点** | 不计入成单转化 | 空调/座椅等不参与电商转化计算 |
| **IVI系统埋点** | 需标注屏幕位置 | main_display/driver_display/hud |
| **ADAS类埋点** | 需标注驾驶模式 | manual/pilot/auto |
| **语音类埋点** | 需标注session | 同一会话共享session_id |

### 3.3 禁止事项

- ❌ 不设计涉及用户隐私敏感信息（如身份证号、完整银行卡号）的埋点
- ❌ 不设计违反数据安全法规的埋点
- ❌ 不为竞品直接复制埋点方案（可作为设计参考）

## 4. 场景分类速查

### 4.1 a级（产品模块）分类规则

```
┌─────────────────────────────────────────────────────────┐
│                    a级分类决策树                         │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  业务类型是？                                           │
│    │                                                    │
│    ├─→ 交易流程 → order_checkout / pay                  │
│    │                                                    │
│    ├─→ 出行服务 → ride_booking / ride_service          │
│    │    (Robotaxi → robotaxi_xxx)                     │
│    │                                                    │
│    ├─→ 智能座舱 → ivi_navi / ivi_entertainment        │
│    │    (ADAS → ivi_adas / ivi_voice)                │
│    │                                                    │
│    ├─→ 用户中心 → user_center / user_profile           │
│    │                                                    │
│    └─→ 其他 → [业务名称]_模块                           │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 4.2 常见混淆场景

| 混淆点 | 正确判断 | 错误示例 |
|-------|---------|---------|
| 空调控制 | 车控 → `ivi_cockpit.ac` | 误归为 `shop` |
| 语音点歌 | 座舱娱乐 → `ivi_entertainment.voice` | 误归为 `shop` |
| 司机叫车 | 出行服务 → `ride_booking` | 误归为 `order_checkout` |
| 车辆预约 | Robotaxi → `robotaxi_booking` | 误归为 `ride_booking` |
| 支付订单 | 支付模块 → `pay.payment` | 误归为 `order_checkout` |

## 5. 进化触发规则

### 5.1 自动触发

| 条件 | 动作 |
|------|------|
| 同一场景执行 ≥3 次 | 生成场景模板 |
| 同一问题反馈 ≥2 次 | 更新检查清单 |
| 新增 SPM ≥5 个 | 提示更新全集 |

### 5.2 手动触发

运行 `reflect` 命令进行深度反思。
