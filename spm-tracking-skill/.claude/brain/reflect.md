## 反思任务

### 目标

阅读过去一周生成的埋点案例和记忆库，总结出业务中最容易混淆的逻辑，并更新到 rules.md 中。

### 执行步骤

#### Step 1: 读取记忆库

```bash
cat .claude/brain/memories.jsonl
```

分析：
- 统计所有任务类型
- 找出被"纠正"的任务
- 识别高频问题

#### Step 2: 读取最近案例

```bash
ls -la data/cases/
cat data/cases/案例名.md
```

分析：
- 总结常见场景模式
- 识别复杂场景

#### Step 3: 分析问题库

```bash
cat data/issues/*.md
```

分析：
- 问题分类统计
- 解决状态

#### Step 4: 生成洞察

回答以下问题：
1. **最常见的错误是什么？**（如：a级命名混淆）
2. **哪些场景最容易出错？**（如：智能座舱/车控类）
3. **用户最常纠正什么？**（如：参数命名）

#### Step 5: 更新规则

根据洞察，更新 `.claude/brain/rules.md`：
- 新增"常见混淆场景"条目
- 更新"业务边界"规则
- 优化"场景分类决策树"

#### Step 6: 记录反思

将反思结果记录到 `data/analytics/reflect_YYYYMMDD.md`

## 自动反思触发

### 触发条件

| 条件 | 触发 |
|------|------|
| 记忆库增长 ≥10 条 | 建议运行 reflect |
| 连续 3 次同类纠正 | 强制运行 reflect |
| 用户手动触发 | /reflect |
