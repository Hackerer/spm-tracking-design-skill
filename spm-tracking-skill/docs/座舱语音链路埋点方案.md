# SPM埋点方案文档

## 1. 需求概述

### 1.1 业务背景

座舱语音助手（Joy）全链路埋点设计，应用于座舱 HMI 屏。基于讯飞星河语音链路方案，融合 JKC 实际业务需求，覆盖从唤醒到休眠的完整语音交互流程。

### 1.2 涉及事件

| 事件 | 阶段 | 说明 |
|------|------|------|
| wakeup_triggered | 阶段1：唤醒激活 | 语音唤醒时触发 |
| audio_session_end | 阶段2：录音与ASR | 用户说话结束/ASR识别完成 |
| nlu_result | 阶段3：意图理解 | 语义理解结果 |
| agent_invoke | 阶段4：业务处理 | 业务指令下发 |
| agent_response | 阶段4：业务处理 | 业务执行结果 |
| tts_result | 阶段5：交互反馈 | TTS播报结果 |
| joy_sleep | 阶段6：休眠 | 语音助手休眠 |

## 2. 公共参数定义

### 2.1 业务公共参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chat_session_id | string | 是 | 对话会话ID，每次唤醒到休眠是一次唯一ID（JKC自己实现） |
| round_id | string | 是 | 单轮交互ID，用户说话到TTS播报结束为一次交互 |
| order_id | string | 是 | 行程订单id |
| user_pin | string | 是 | 用户京东pin |

#### 2.1.1 chat_session_id 详细定义

**定义**：从用户唤醒语音助手，到语音助手休眠，整个过程为一次唯一会话。

**生命周期**：唤醒 → 休眠

**生成规则**：
```
chat_session_id = "jkc_{deviceId}_{timestamp}_{random}"
// 示例：joy_vin123_1712999999999_a1b2c3
```

**生成时机**：

| 场景 | 是否生成新session |
|------|------------------|
| 正常唤醒 → 休眠 | 是 |
| 重新唤醒（未休眠） | 是 |
| 多模唤醒 | 是 |
| 抢麦唤醒 | 是 |
| 定时自动休眠 | 销毁当前session |

**星河对应**：dialogueID

#### 2.1.2 round_id 详细定义

**定义**：用户说一句话 + 助手回复一句话 = 一轮交互

**生命周期**：用户开始说话 → TTS播报结束

**生成规则**：
```
round_id = "round_{sessionId}_{increment}"
// 示例：round_jkc_vin123_1712999999999_a1b2c3_001
```

**交互示例**：

```
round_1: 用户"你好" → 助手"在呢"
round_2: 用户"把温度调到18度" → 助手"好的，温度已调到18度"
round_3: 用户"再打开车窗" → 助手"车窗已打开"
```

**对话种类与round关系**：

| 对话种类 | session变化 | round变化 |
|----------|-------------|-----------|
| 单轮对话 | 1个session | 1个round |
| 多轮对话 | 1个session | N个round |
| 免唤醒对话 | 不变 | 新round |
| 主动对话 | 不变 | 新round |

**星河对应**：interactiveID

## 3. 时间点定义

| 时间点 | 参数名 | 说明 |
|--------|--------|------|
| T1 | start_called | 会话开始时间（人声第一帧时间点） |
| T2 | get_first_word | 获取首字结果时间 |
| T3 | typewriter_first_show_time | 打字机首次展示文字的时间点 |
| T5 | user_voice_end_time | 用户结束说话的时间点 |
| T6 | os_last_word_time | 端上回调打字机末字消息时间 |

### 时间戳计算指标

| 指标 | 计算方式 | 指标值 |
|------|----------|--------|
| 唤醒UI响应时长 | awaken_ui_duration = T3 - T1 | 260ms ± 50 |
| 唤醒TTS播报时长 | awaken_tts_duration = T2 - T1 | < 100ms |
| 打字机首字上屏耗时 | typewriter_first_word_duration = T3 - T1 | 900ms |
| 打字机响应耗时 | typewriter_response_duration = T6 - T5 | 1200ms |
| TTS响应时长 | tts_response_duration | 用户说话结束到第一帧播报 |

## 5. 语音执行成功率链路

### 5.1 端到端成功率指标

```
意图理解(nlu_result) → 业务指令下发(agent_invoke) → 业务执行(agent_response) → TTS回复(tts_result)
```

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| 意图识别率 | NLU正确理解用户意图的比例 | is_unknow=否的数量 / 总ASR数 |
| 指令下发成功率 | 业务指令成功下发的比例 | command_status=success / 意图识别成功数 |
| 执行成功率 | 业务指令执行完成的比例 | execute_status=success / 下发成功数 |
| 端到端成功率 | 完整语音操作成功的比例 | tts_result=成功 / 总请求数 |

### 5.2 失败原因枚举

| fail_reason | 说明 |
|-------------|------|
| timeout | 业务处理超时 |
| device_error | 设备异常 |
| param_error | 参数错误 |
| network_error | 网络异常 |
| service_error | 服务异常 |
