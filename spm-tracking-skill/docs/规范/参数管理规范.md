# 参数管理规范

## 文档概述

本文档定义埋点参数的统一管理规范，包括参数类型定义、校验规则、公共参数标准等。

---

## 一、参数类型定义

### 1.1 基础类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **string** | 字符串类型 | "abc", "pending", "product_123" |
| **number** | 数值类型 | 123, 99.00, 1000 |
| **boolean** | 布尔类型 | true, false |
| **array_string** | 字符串数组 | ["a", "b", "c"] |
| **array_number** | 数值数组 | [1, 2, 3] |
| **object** | 对象类型 | {"key": "value"} |

### 1.2 特殊类型

| 类型 | 说明 | 格式要求 |
|------|------|----------|
| **id类** | 唯一标识符 | string类型，通常为数字或字母组合 |
| **时间类** | 时间戳 | number类型，单位毫秒/秒 |
| **金额类** | 金额 | number类型，单位统一（如分） |
| **枚举类** | 固定值集合 | string类型，只能在枚举范围内取值 |

## 二、校验规则定义

### 2.1 校验规则类型

| 规则类型 | 说明 | 适用类型 |
|----------|------|----------|
| **length** | 长度校验 | string, array |
| **enum** | 枚举校验 | string, number |
| **regex** | 正则校验 | string |
| **range** | 范围校验 | number |
| **required** | 必填校验 | all |

### 2.2 校验规则示例

```json
{
  "参数名": "product_id",
  "类型": "string",
  "必填": true,
  "校验规则": {
    "length": {
      "min": 1,
      "max": 50
    }
  }
}

{
  "参数名": "price",
  "类型": "number",
  "必填": true,
  "校验规则": {
    "range": {
      "min": 0,
      "max": 99999999
    }
  }
}

{
  "参数名": "status",
  "类型": "string",
  "必填": true,
  "校验规则": {
    "enum": ["pending", "success", "fail", "cancel"]
  }
}

{
  "参数名": "phone",
  "类型": "string",
  "必填": false,
  "校验规则": {
    "regex": "^1[3-9]\\d{9}$"
  }
}
```

## 五、参数命名规范

### 5.1 命名原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 语义化 | 使用有意义的英文单词 | `product_id` 而非 `p_id` |
| 小写下划线 | 全部小写，单词间用下划线 | `order_id`, `total_amount` |
| 避免缩写 | 除非是公认缩写 | `id`, `url`, `api` 可用 |
| 保持一致 | 同类参数使用统一命名 | 所有商品ID都用 `product_id` |

### 5.2 常见参数命名

| 参数含义 | 推荐命名 | 不推荐 |
|----------|----------|--------|
| 商品ID | product_id | pid, goods_id, item_id |
| SKU ID | sku_id | skuid, skuId |
| 订单ID | order_id | oid, orderId |
| 用户ID | user_id | uid, userId |
| 分类ID | category_id | cid, cat_id |
| 品牌ID | brand_id | bid |
| 价格 | price | amount, money |
| 数量 | quantity | num, count |
| 状态 | status | state, s |
| 时间 | timestamp | time, ts |

## 七、参数管理流程

