# SPM埋点方案文档

## 1. 需求概述

### 1.1 业务背景
健康监测 HMI 车机版埋点设计，应用于座舱 HMI 屏。用户通过健康监测应用进行性别选择、基础信息录入、协议确认、教学引导、检测准备、监测过程、报告生成和结果查看等完整流程。

### 1.2 涉及页面
| 页面 | 页面SPM | 说明 |
|------|---------|------|
| 性别选择页 | health_monitor.gender_selection | 选择性别后进入基础信息录入 |
| 基础信息录入页 | health_monitor.basic_info | 录入年龄、身高、体重 |
| 健康监测启动页 | health_monitor.start | 价值介绍、协议勾选、一键开启 |
| 协议详情弹窗 | health_monitor.agreement_detail | 知情同意书内容 |
| 隐私政策弹窗 | health_monitor.privacy_policy | 隐私政策内容 |
| 教学引导页 | health_monitor.guide | 取出设备、双手持握指引 |
| 检测准备页 | health_monitor.prepare | 信号采集、准备状态 |
| 监测过程页 | health_monitor.monitoring | 监测进度、实时波形 |
| 退出确认弹窗 | health_monitor.exit_confirm | 中途退出挽留 |
| 报告生成页 | health_monitor.report_generating | 报告生成中等待 |
| 结果报告页 | health_monitor.report_result | 总评分、生理/心理指标 |

## 3. 参数定义（统一）

### 3.1 公共业务参数 (HMI业务公参)
| 参数名 | 类型 | 必填 | 说明 | 枚举值 |
|--------|------|------|------|--------|
| user_pin | string | 是 | 用户京东pin | - |
| navi_bar_status | string | 是 | 导航栏状态 | 沉浸式/非沉浸式 |
| vid | string | 是 | 车辆id | - |
| server_env | string | 是 | 服务端环境 | live/staging/test |
| build_type | string | 是 | 包环境类型 | 可切环境包/不可切环境包 |
| order_id | string | 是 | jkc订单id | - |
| click_type | string | 是 | 点击触发方式 | voice/touch/gesture |

### 3.2 事件特有参数
| 参数名 | 类型 | 说明 | 枚举值 |
|--------|------|------|--------|
| gender | string | 性别 | male/female |
| age | number | 年龄 | - |
| height | number | 身高(cm) | - |
| weight | number | 体重(kg) | - |
| agreement_type | string | 协议类型 | - |
| monitor_id | string | 监测ID | - |
| content_id | string | 内容ID | - |
