需求概述
需求背景
座舱本质上提供了一个高粘性、高专注度的黄金健康场景——用户出行过程中被“锁定”在这个封闭空间内，既无法像手机端那样快速划走，也不会像居家场景那样被琐事打断，较为适合深度健康服务渗透。同时，出行场景具备高频与周期性的属性，高频的用车行为创造了一个天然的健康管理入口：以通勤族为例，用户平均每天在车内停留30-50分钟，每周重复5-7次，一方面能为健康业务提供稳定的线下流量，为交易转化提供基础；另一方面，周期性的监测在积累数据资产的同时，也能培养用户习惯，提升用户对出行服务的粘性。
基于此，希望在座舱内接入轻量化健康监测服务，通过高频沉浸式场景与周期监测的组合，让健康服务从“偶发事件”升级为“生活习惯”，提升出行服务粘性的同时，实现座舱内的商业化变现。
需求价值
用户价值：行程空闲时提供放手可测的服务，解决有健康焦虑但「忘记测」「没空测」的问题
「2025年Q1全球可穿戴腕带设备同比增长13%，健康追踪是购买时最看重的三大功能之一」
业务价值：通过附加服务提升用户打车粘性，同时为健康服务引流，作为触点激发健康侧转化
「出行需求难以直接被附加服务激发，但可通过该方式在存量需求中提升用户对我们的粘性」
评价指标
功能渗透率 = 行程中开启该功能的行程数 / 有效行程数
使用完成率 = 行程中使用完成该功能的行程数 / 行程中开启该功能的行程数

用户点击入口进入到健康监测页面后，展示监测启动页，页面内对监测内容及监测关键步骤进行介绍

开启监测按钮：用户点击开启监测按钮后，执行下述校验：
监测须知校验：进入页面后默认勾选；用户点击按钮后，校验监测须知是否已勾选，若未勾选，则toast提示“请先阅读并同意监测须知”；若已勾选，则校验通过；
协议详情：协议共两份，用户点击文字链后即可进入协议详情页；协议详情页展示协议信息（服务端配置），页面提供“不同意”与“同意并继续”按钮，用户点击“不同意”后返回启动页并取消知情同意的勾选；用户点击“同意并继续”则返回启动页并保持知情同意的勾选
系统信息层级：底部dock栏、顶部行程状态栏均保留（后续页面相同）；监测全流程中，用户均可点击dock栏进行交互
系统交互
行程时间校验：用户点击“开启监测”按钮后，查询当前订单的预估剩余时间，若当前预估剩余时间不足3分钟，则toast提示“行程即将结束，欢迎下次再尝试”；若预估时间≥3分钟，则允许开启监测。
页面流转：用户点击“开启监测”按钮且页面校验通过后，即流转至教学引导页面
监测ID获取：用户点击“立即监测”按钮后，客户端请求服务端，由JKC服务端请求健康服务端获取监测ID；取当前占用该车辆的订单所属的pin作为入参，上送至健康侧。
监测启动信号下发：获取到监测ID后，即开始回收传感器模组采集的ECG信号
页面流转：获取到监测ID后，即跳转至监测引导页面
引导视频：页面左侧展示引导文案，右侧展示教学引导动画
准备进度条：用户需持握传感器满2s（信号稳定上报）后，才可进入后续监测；页面展示准备进度条，为用户进行接触反馈，表达接触状态是否持续正常，达到开启监测的标准；具体展示及校验逻辑如下：
客户端在进入该页面后即启动蓝牙数据回收，并判断是否有正常数据回收
若30s内未回收到上报数据，则弹窗提示接触异常；
若回收到有效数据，则启动2s（可配置）计时，以2s为时间周期，判断信号是否稳定上报；
若回收到有效信号，则状态条启动；状态条按2s一周匀速转动，若中途信号中断，则状态条迅速弹回初始状态；直至2s时间周期完整上报后，左侧文案区提示用户即将开始监测，请稍后；页面停留2s（可配置）后即跳转至监测过程页。
同时，回收到有效信号后，教学引导视频不再循环播放，播放完当次的视频即停止并定格在最后一帧（双手持握）；即使后续用户脱手也不再重新播放，仅当用户再次返回该页面（如后续流程中脱手，选择重新开始检测）后才重新播放。
监测进度提示：页面左侧展示监测进度及指标动效
时间进度：根据倒计时 60s换算百分比，采用平方根函数进行百分比映射，即进度百分比=sqrt（已计时的时间/90）*100；实现进度前快后慢，减轻用户在等待过程中的焦虑感

<script>
  const totalTime = 60; // 总时间，单位秒
  const progressBar = document.getElementById('progressBar');
  const progress = document.getElementById('progress');

  function updateProgress(percentage) {
    progress.style.width = `${percentage}%`;
  }

  function nonLinearMapping(timeElapsed) {
    // 使用平方根函数作为非线性映射
    return Math.sqrt(timeElapsed / totalTime) * 100;
  }

  function startCountdown() {
    let startTime = Date.now();
    const interval = setInterval(() => {
      const now = Date.now();
      const timeElapsed = (now - startTime) / 1000; // 转换为秒

      if (timeElapsed >= totalTime) {
        clearInterval(interval);
        updateProgress(100); // 倒计时结束时设置进度为100%
      } else {
        const progressPercentage = nonLinearMapping(timeElapsed);
        updateProgress(progressPercentage);
      }
    }, 100); // 每100毫秒更新一次
  }

  startCountdown();
</script>
