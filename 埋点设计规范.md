# 埋点设计规范

## 文档概述

本文档定义通用的埋点设计规范，基于京东SPM（Super Position Model）位置模型标准，适用于各业务线的埋点方案设计。

---

## 一、SPM位置模型定义

### 1.1 SPM四级结构

SPM（Super Position Model）位置模型是用于用户行为埋点和流量追踪的核心模型，通过层级化的编码结构，唯一标识页面中的每一个可交互元素的位置。

```
SPM = a.b.c.d

a级：产品/业务线 → 业务域或频道
b级：页面/模块   → 具体的页面
c级：区块/区域   → 页面中的功能区域
d级：坑位/元素   → 区域内的具体交互元素
```

### 1.2 各层级定义

| 层级 | 名称 | 说明 | 示例 |
|------|------|------|------|
| a | 产品/业务线 | 表示一个产品或频道，包含一组页面 | shop, cart, user, pay, search |
| b | 页面/模块 | 标识具体页面 | product_detail, checkout, order_list |
| c | 区块 | 页面中的某个区块或功能区域 | banner, recommend, list, nav |
| d | 坑位 | 区块内的具体交互元素 | buy_btn, add_cart, favorite |

### 1.3 坑位类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 固定坑位 | 固定不变的可交互位置 | 各种功能操作按钮、标题、分享按钮 |
| 顺序坑位 | 承载可变内容的顺序排列位置 | feeds流卡片、金刚区、算法控制的tab |

## 二、事件类型定义

### 2.1 事件分类

| 事件类型 | 说明 | 触发时机 |
|----------|------|----------|
| **浏览(PV)** | 铺满整个屏幕的页面，或存在相同功能的半弹层 | 进入时、回退时、后台回来时、亮屏时 |
| **曝光** | 页面元素展示在屏幕中，真实被用户看到 | 元素漏出时、元素漏出50%且停留300ms |
| **点击** | 用户点击或滑动选中页面元素 | 点击或滑动选中、接口返回/状态变化 |
| **订单** | 用户点击下单（点击的一种，重要） | 点击下单按钮，接收服务端返回订单ID后 |
| **技术** | 用于技术监控 | 自定义时机 |

### 2.2 事件采集规范

#### 浏览事件（PV）
- 页面进入时上报
- 回退到当前页时上报
- 从后台回来时上报
- 息屏再亮屏时上报
- Tab切换完全漏出时上报（APP和通天塔页面不上报，小程序和H5上报）
- 下拉刷新时不上报

#### 曝光事件
- 元素首次漏出时触发
- 元素漏出50%且停留300ms触发（商卡类）
- 页面滑动反复漏出无需上报
- 页面返回、刷新数据曝光需要再次上报

#### 点击事件
- 所有用户有效点击都需要上报
- 接口返回或业务状态发生变化时触发

## 三、SPM命名规范

### 3.1 命名原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 语义化 | 使用有意义的英文单词或缩写 | `product_detail` 而非 `p_d` |
| 小写下划线 | 全部小写，单词之间用下划线分隔 | `buy_button` |
| 层级递进 | 从a到d层级应该逻辑递进 | `shop → product_detail → banner → buy_btn` |
| 唯一性 | 同一个页面的同一个位置使用唯一SPM | 不重复 |

### 3.2 各层级命名建议

#### a级（产品/业务线）
```
shop      - 购物模块
cart      - 购物车模块
user      - 用户模块
pay       - 支付模块
search    - 搜索模块
recommend - 推荐模块
```

#### b级（页面/模块）
```
index           - 首页
product_list    - 商品列表页
product_detail  - 商品详情页
checkout        - 结算页
order_confirm   - 订单确认页
order_list      - 订单列表页
user_center     - 用户中心
```

#### c级（区块）
```
banner          - 顶部轮播/横幅
recommend       - 推荐区域
list            - 列表区域
nav             - 导航区域
fixed_bottom    - 底部固定区域
fixed_top       - 顶部固定区域
```

#### d级（坑位）
```
buy_btn         - 立即购买按钮
add_cart        - 加入购物车按钮
favorite        - 收藏按钮
share           - 分享按钮
view_more       - 查看更多
```

## 四、业务类型（biz_type）规范

### 4.1 适用场景

当d级坑位内存在多个不同类型的可交互位置时，需要使用`biz_type`（业务类型）进行细分描述。

### 4.2 常见业务类型

| biz_type | 说明 | 适用场景 |
|----------|------|----------|
| card | 商卡/商品卡片 | 商品展示类 |
| add_cart | 加购 | 加购按钮 |
| cart | 购物车 | 购物车入口 |
| shop | 店铺入口 | 店铺相关 |
| rank | 榜单 | 排行榜入口 |
| coupon | 优惠券 | 优惠券相关 |
| live | 直播 | 直播入口 |
| video | 视频 | 视频内容 |

### 4.3 组合使用

```
SPM: shop.product_detail.feeds.card
biz_type: card

SPM: shop.product_detail.feeds.card
biz_type: add_cart

SPM: shop.product_detail.feeds.card
biz_type: coupon
```

## 五、事件埋点示例

### 5.1 完整埋点结构

```
SPM: shop.product_detail.banner.buy_btn
事件类型: 点击
业务类型: add_cart
触发时机: 点击立即购买按钮
```

### 5.2 埋点清单示例

| 序号 | 事件ID | 事件名称 | SPM | 事件类型 | biz_type | 触发时机 |
|------|--------|----------|-----|----------|----------|----------|
| 1 | EVT001 | 商品曝光 | shop.product_detail.feeds.card | 曝光 | card | 商品卡片漏出50%且停留300ms |
| 2 | EVT002 | 商品点击 | shop.product_detail.feeds.card | 点击 | card | 点击商品卡片 |
| 3 | EVT003 | 立即购买点击 | shop.product_detail.banner.buy_btn | 点击 | buy | 点击立即购买按钮 |
| 4 | EVT004 | 加入购物车 | shop.product_detail.fixed_bottom.add_cart | 点击 | add_cart | 点击加入购物车 |
| 5 | EVT005 | 收藏点击 | shop.product_detail.info.favorite | 点击 | favorite | 点击收藏按钮 |

## 六、SPM设计原则

### 6.1 核心原则

1. **层级清晰**：a→b→c→d 层级逻辑递进，不能跳跃
2. **全局唯一**：同一产品线内，SPM编码应该唯一
3. **语义明确**：命名应该能够表达页面/区块/元素的功能
4. **继承关系**：子层级应该继承父层级的语义
5. **可扩展性**：预留顺序坑位，应对内容变化

### 6.2 设计检查清单

- [ ] a级是否符合业务线划分？
- [ ] b级是否对应具体页面？
- [ ] c级是否标识了页面区域？
- [ ] d级是否唯一标识了交互元素？
- [ ] 是否有重复的SPM编码？
- [ ] 命名是否清晰易懂？
- [ ] 顺序坑位是否预留？
